# ARHOMUS — пошаговая сборка сайта на Tilda (старт с уже добавленными T123 и T100)

> Входные данные: у вас уже есть страница с двумя блоками — `T123 (HTML)` и `T100 (корзина)`.
> Цель: собрать рабочий магазин с карточками товаров, корзиной и двуязычностью RU/EN.

---

## 0) Что сделать до сборки (10 минут)

1. Подготовьте 2 CSV-файла из репозитория:
   - `data/arhomus_catalog_seed.csv` (RU)
   - `data/arhomus_catalog_seed_en.csv` (EN)
2. Подготовьте медиа (фото/видео) по каждому товару.
3. Проверьте, что логотипы готовы в светлой и темной версии.

---

## 1) Базовая структура проекта в Tilda

### Вариант запуска (рекомендованный)
- Создайте **2 проекта**:
  - `ARHOMUS RU`
  - `ARHOMUS EN`

Почему так проще: быстрее поддержка, меньше риска перепутать языковой контент на MVP.

### Какие страницы создать в каждом проекте
1. `/` — Главная
2. `/shop` — Каталог
3. `/cart` — Корзина/оформление
4. `/product` — (служебные страницы товара, если используете отдельную карточку)

---

## 2) Что делать на текущей странице с T123 + T100

У вас уже добавлены два блока. Теперь:

1. Откройте блок **T123** → «Контент».
2. Вставьте код из файла `tilda/arhomus_t123_theme_snippet.html` целиком.
3. Сохраните.
4. Проверьте блок **T100** (корзина):
   - включен на странице,
   - виден на опубликованной версии,
   - не скрыт в настройках видимости.

> Если в вашем интерфейсе корзина называется `ST100`, это тот же целевой блок для магазина.

---

## 3) Подключение каталога товаров

1. В проекте откройте **Товары / Product Catalog**.
2. Нажмите **Импорт**.
3. Для RU-проекта импортируйте `data/arhomus_catalog_seed.csv`.
4. Для EN-проекта импортируйте `data/arhomus_catalog_seed_en.csv`.
5. Проверьте после импорта:
   - цена (`Price`),
   - остаток (`Quantity`),
   - SKU,
   - поле готовности отправки (`ReadyToShip`).

---

## 4) Вывод товаров на страницу каталога (`/shop`)

1. Добавьте Store-блок каталога (любой блок карточек товаров из раздела Store).
2. В «Контенте» блока выберите источник: **Product Catalog**.
3. Настройте сетку карточек (например 3 в ряд на desktop, 2 на tablet, 1 на mobile).
4. Включите в карточке отображение:
   - Названия,
   - Цены,
   - Кнопки «Купить» / Add to cart.

---

## 5) Настройка карточки товара

Для каждого товара в Product Catalog заполните:
1. Название
2. Описание (единый шаблон)
3. Цена
4. Остаток
5. Срок отправки: `1 день`
6. Фото и/или видео

### Рекомендуемая структура описания
- Блок 1: Идея / Манифест
- Блок 2: Дизайн (Tim Viter)
- Блок 3: Материалы и конструкция
- Блок 4: Упаковка

---

## 6) Корзина и базовый checkout (на дефолте)

Сейчас делаем простой рабочий MVP:

1. Оставляем корзину на стандартных настройках Tilda.
2. В «Настройки сайта → Платежные системы» пока ничего не усложняем.
3. В «Настройки сайта → Доставка» оставляем дефолтный сценарий.
4. Включаем уведомления о заказах на нужный email.

Позже можно подключить эквайринг/доставку в деталях без переделки дизайна.

---

## 7) Двуязычность: переключение RU/EN

1. В header добавьте 2 ссылки:
   - `RU` → ссылка на RU-версию
   - `EN` → ссылка на EN-версию
2. Примените CSS-класс `arh-lang-switch` (он уже есть в T123-сниппете).
3. На RU- и EN-проектах проверьте, что ссылки взаимно ведут на нужные страницы.

---

## 8) Проверка перед публикацией (чеклист)

### Функционал
- Добавление товара в корзину работает.
- Количество в корзине меняется.
- Итоговая сумма считается корректно.
- Форма заказа отправляется.

### Контент
- В карточках правильные цены.
- В карточках правильные остатки.
- Для всех in-stock товаров указан срок `1 день`.
- EN-тексты действительно на английском.

### Адаптив
- Проверить 320 / 375 / 768 / 1024 / 1440.

---

## 9) Порядок публикации

1. Публикуете RU-проект.
2. Прогоняете 1 тестовый заказ.
3. Публикуете EN-проект.
4. Прогоняете 1 тестовый заказ.
5. Только после этого включаете рекламу/трафик.

---

## 10) Частые проблемы и быстрые решения

1. **Стили из T123 не применились**
   - Проверьте, что страница опубликована (не только просмотр в редакторе).
2. **Товары не появляются в блоке каталога**
   - Проверьте источник данных блока: должен быть Product Catalog.
3. **Корзина не показывает товары**
   - Убедитесь, что на странице есть блок T100/ST100 и кнопка карточки связана с магазином.
4. **RU/EN путаются**
   - Проверьте отдельные CSV на каждом проекте и ссылки переключателя.

## 12) Где именно «точная копия» главной и как вставить

Точный шаблон (на основе загруженного webarchive донор-сайта) подготовлен в файле:
- `tilda/arhomus_vivol_exact_template.html`

Как вставить:
1. Откройте главную страницу в Tilda.
2. Откройте ваш блок `T123`.
3. Полностью замените содержимое T123 на код из `tilda/arhomus_vivol_exact_template.html`.
4. Опубликуйте страницу.

Перед вставкой (рекомендовано):
1. В терминале запустите автоподготовку шаблона:
   - `python scripts/retarget_vivol_template.py`
2. Скрипт возьмет исходный webarchive и пересоберет `tilda/arhomus_vivol_exact_template.html` с заменой бренда и donor-изображений на ARHOMUS-ассеты из репозитория.


### Если вы на iPad и без терминала
Ничего запускать не нужно.

Используйте уже готовый файл из репозитория:
- `tilda/arhomus_vivol_exact_template.html`

Шаги на iPad:
1. Откройте GitHub/файловое хранилище, где лежит репозиторий.
2. Откройте файл `tilda/arhomus_vivol_exact_template.html`.
3. Нажмите `Raw` (или откройте полный текст файла) и скопируйте весь HTML.
4. В Tilda откройте страницу → блок `T123` → вставьте HTML целиком.
5. Сохраните и опубликуйте страницу.

Скрипт `python scripts/retarget_vivol_template.py` нужен только если вы хотите пересобирать шаблон автоматически после новых правок в webarchive.

Что это дает:
- Основа страницы будет максимально близка к донору `vivol.store`.
- В шаблоне уже выполнены базовые замены бренда `Vivol` → `ARHOMUS` в title/мета и текстовых вхождениях.

Важно:
- Для живых товарных карточек и корректной корзины оставьте `/shop` на Store-блоках + Product Catalog.
- Если какой-то контент (изображения/иконки) тянется с внешних URL донора, его нужно поэтапно заменить на ваши ассеты в Tilda (загрузить в проект и подставить ссылки).
